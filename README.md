## ОТВЕТ:

Чтобы определить типы портов (Root, Designated, Blocked/Non-Designated), нужно учесть, что корневым коммутатором является SW1.

Важное допущение: В условии сказано «Priority порт у всех портов по умолчанию». В реальной работе STP приоритет порта (Cost) зависит от скорости. Я буду использовать стандартные значения стоимости (Cost) для 802.1D (указаны на вашей схеме в легенде):

· 100 Mbps (fa0/x) = Cost 19 \
· 1 Gbps (ge0/x) = Cost 4 \
· 10 Gbps (te0/x) = Cost 2 \

Поскольку у SW1 и SW2 есть канал 10 Гбит/с (te0/1), а у остальных максимум 1 Гбит/с, это сильно влияет на маршруты.

1. Корневые порты (Root Ports - RP)

Это порты на некорневых коммутаторах, которые обеспечивают кратчайший путь к корневому коммутатору (SW1).

· SW2: Выбирает между te0/1 (Cost 2) и ge0/2 (Cost 4). te0/1 (к SW1) — Root Port. \
· SW3: Выбирает между ge0/1 (к SW1, Cost 4) и fa0/3 (к SW4, Cost 19). ge0/1 — Root Port. \
· SW4: Имеет 3 пути. Считаем стоимость до SW1: \
  · Через fa0/3 (SW4) -> fa0/3 (SW3) -> ge0/1 (SW3): 19 + 4 = 23. \
  · Через ge0/1 (SW4) -> te0/2 (SW2) -> te0/1 (SW2): 4 + 2 = 6. \
  · Через ge0/2 (SW4) -> ge0/2 (SW5) -> ge0/1 (SW5) -> te0/2 (SW2) -> te0/1 (SW2): 4 + 4 + 4 + 2 = 14. \
  · ge0/1 (в сторону SW2) — Root Port (Cost 6). \
· SW5: Два пути к SW1: \
  · Через ge0/1 (SW5) -> te0/2 (SW2) -> te0/1 (SW2): 4 + 2 = 6. \
  · Через ge0/2 (SW5) -> ge0/2 (SW4) -> ge0/1 (SW4) -> te0/2 (SW2) -> te0/1 (SW2): 4 + 4 + 4 + 2 = 14. \
  · ge0/1 (в сторону SW2) — Root Port. \
· SW6: Путь к SW1: fa0/1 -> fa0/2 (SW3) -> ge0/1 (SW3). Cost: 19 + 4 = 23. fa0/1 — Root Port. \
· SW7: Путь к SW1: fa0/1 -> fa0/4 (SW3) -> ge0/1. Cost: 19 + 4 = 23. fa0/1 — Root Port. \
· SW8: Путь к SW1: fa0/1 -> fa0/4 (SW4) -> ge0/1 (SW4) -> te0/2 (SW2) -> te0/1 (SW2). Cost: 19 + 4 + 2 = 25. fa0/1 — Root Port. \
· SW9: Путь к SW1: fa0/1 -> fa0/3 (SW5) -> ge0/1 (SW5) -> te0/2 (SW2) -> te0/1 (SW2). Cost: 19 + 4 + 2 = 25. fa0/1 — Root Port. \
· SW10: Путь к SW1: fa0/1 -> fa0/2 (SW6) -> fa0/1 (SW6) -> fa0/2 (SW3) -> ge0/1 (SW3). Cost: 19 + 19 + 19 + 4 = 61. fa0/1 — Root Port.

---

2. Назначенные порты (Designated Ports - DP)

Это порты, которые отправляют лучшие BPDU на сегменте. Обычно это порты на стороне коммутатора с наименьшей стоимостью до корня (ближе к корню) или сам корневой коммутатор.

· Корневой SW1: Все его порты (te0/1, ge0/1, fa0/3) являются Designated Ports. \
· SW2: Порты te0/2 (в сторону SW4) и ge0/2 (в сторону SW5) являются Designated Ports, так как SW2 ближе к корню, чем SW4 и SW5. \
· SW3: Порты fa0/2 (к SW6), fa0/4 (к SW7) и fa0/3 (к SW4) являются Designated Ports. (Примечание: Хотя fa0/3 соединен с SW4, SW3 имеет стоимость 4 до корня, а SW4 — 6, поэтому побеждает SW3). \
· SW6: Порт fa0/2 (в сторону SW10) является Designated Port. \
· SW4: Порт fa0/4 (в сторону SW8) является Designated Port. \
· SW5: Порт fa0/3 (в сторону SW9) является Designated Port.

---

3. Неназначенные / Блокирующие порты (Blocked / Non-Designated Ports)

Это порты, которые останавливают петли. Они не выбираются как Root или Designated.

· SW4 — fa0/3: Этот порт смотрит на SW3. SW3 — Designated на этом кабеле (так как он ближе к корню). Значит, порт на SW4 блокируется. (Blocked) \
· SW5 — ge0/2: Этот порт смотрит на SW4. SW4 ближе к корню (Cost 6), чем SW5 (Cost 6, но проигрывает по сравнению с SW4 из-за BPDU, так как они на одной стоимости, сравнивается Bridge ID: SW4 (MAC D) < SW5 (MAC E)). Поэтому порт ge0/2 на SW5 блокируется. (Blocked)

Итоговый список состояний портов (для всех коммутаторов):

· SW1 (Корень): Все DP (te0/1, ge0/1, fa0/3). \
· SW2: te0/1 (RP), te0/2 (DP), ge0/2 (DP). \
· SW3: ge0/1 (RP), fa0/2 (DP), fa0/3 (DP), fa0/4 (DP). \
· SW4: ge0/1 (RP), fa0/3 (Blocked), fa0/4 (DP), ge0/2 (DP). \
· SW5: ge0/1 (RP), ge0/2 (Blocked), fa0/3 (DP). \
· SW6: fa0/1 (RP), fa0/2 (DP). \
· SW7: fa0/1 (RP). \
· SW8: fa0/1 (RP). \
· SW9: fa0/1 (RP). \
· SW10: fa0/1 (RP).
